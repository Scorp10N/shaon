**Dispatch mechanics:**
- *MC-agent path:* `mc_create_task` each row above with `assigned_to` set to
  whichever registered MC agent (`claude-code`, `codex`, etc. — see
  `mc_list_agents`) should own it; update `status`/`assigned_to` via
  `mc_update_task` as work progresses.
- *Sub-agent path (agent-agnostic, this session):* use the `Agent` tool with
  `subagent_type` appropriate to a coding task, one dispatch per plan task,
  following `superpowers:subagent-driven-development`'s fresh-subagent-per-task
  pattern — each sub-agent gets the exact task text from this plan file as
  its prompt, since every task here is already self-contained (exact file
  paths, exact code, exact commands) per `writing-plans`' no-placeholder rule.
- Either path can be mixed: e.g. Task 1 to an MC-registered `codex` agent
  while Tasks 3/4/5 run as three parallel Claude Code sub-agents in this
  session — they don't conflict since they touch disjoint files/repos.

**Goal:** Decouple `shaon` from real Hilan credentials by (a) patching `shaon` to optionally talk to a proxy URL instead of the real Hilan domain, and (b) building the `authveil` proxy's non-MFA foundation (reverse proxy core + Hilan password-only login, reusing `provider-hilan`) plus a standalone, fully-testable TOTP generator. Wiring TOTP into the actual Hilan login flow is **not** in this plan — it depends on capturing Hilan's real MFA wire format first (Task 6 here produces that capture as a data artifact; the follow-up plan consumes it).

**Architecture:** `shaon`'s `HilanClient` gets an optional `proxy_url` override so it can point at any base URL and skip its own login path entirely. `authveil` is a new Rust binary crate depending on `provider-hilan` (git dep, pinned to this fork's commit) that runs an axum HTTP server: it forwards arbitrary requests to a configurable upstream host while owning its own authenticated `HilanClient` session in-process.

**Tech Stack:** Rust, axum (reverse-proxy HTTP server), reqwest (outbound forwarding — already a workspace dep via `provider-hilan`), tokio, `provider-hilan` (reused login/cookie logic), no new TOTP crate — hand-rolled RFC 6238 (HMAC-SHA1) since it's ~15 lines and this avoids an extra dependency for something with official, verifiable test vectors.

## Global Constraints

- Rust 1.80+ (matches `shaon`'s workspace requirement, `Cargo.toml:9`).
- No new mocking/HTTP-mock crate for tests — reuse the existing raw-`TcpListener` mock-server pattern already established in `crates/provider-hilan/src/client.rs` (`spawn_test_server`, `crates/provider-hilan/src/client.rs:1697`), for consistency and to avoid adding a dependency this codebase doesn't already use.
- `authveil` never persists a real Hilan password or TOTP seed to its own disk in this plan — secrets arrive via env vars in these tasks (the sops/age/tmpfs wiring from the design spec is a deployment-time concern, out of scope for this code-level plan).
- Every task must leave `cargo build --workspace` and `cargo test --workspace` green in the repo it touches before its commit step.

```rust
use base32::Alphabet;
use hmac::{Hmac, Mac};
use sha1::Sha1;
use std::fmt;

const STEP_SECONDS: u64 = 30;
const DIGITS: u32 = 6;

#[derive(Debug)]
pub enum TotpError {
    InvalidSeed,
}

impl fmt::Display for TotpError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            TotpError::InvalidSeed => write!(f, "seed is not valid base32"),
        }
    }
}

impl std::error::Error for TotpError {}

/// Generate an RFC 6238 TOTP code for `base32_seed` at `unix_time`.
pub fn generate_totp(base32_seed: &str, unix_time: u64) -> Result<String, TotpError> {
    let key = base32::decode(Alphabet::Rfc4648 { padding: false }, base32_seed)
        .ok_or(TotpError::InvalidSeed)?;
    let counter = unix_time / STEP_SECONDS;
    let counter_bytes = counter.to_be_bytes();

    let mut mac = Hmac::<Sha1>::new_from_slice(&key).map_err(|_| TotpError::InvalidSeed)?;
    mac.update(&counter_bytes);
    let hash = mac.finalize().into_bytes();

    let offset = (hash[hash.len() - 1] & 0x0f) as usize;
    let binary = ((hash[offset] as u32 & 0x7f) << 24)
        | ((hash[offset + 1] as u32) << 16)
        | ((hash[offset + 2] as u32) << 8)
        | (hash[offset + 3] as u32);
    let code = binary % 10_u32.pow(DIGITS);

    Ok(format!("{code:0width$}", width = DIGITS as usize))
}

#[cfg(test)]
mod tests {
    use super::*;

    // RFC 6238 Appendix B test vectors use the ASCII string
    // "12345678901234567890" as the SHA1 seed, base32-encoded here.
    const RFC6238_SHA1_SEED_B32: &str = "GEZDGNBVGY3TQOJQGEZDGNBVGY3TQOJQ";

    #[test]
    fn matches_rfc6238_test_vector_at_time_59() {
        // RFC 6238 Appendix B: T=59, SHA1 -> "94287082"; this crate emits
        // 6-digit codes (DIGITS=6), so compare against the low 6 digits
        // convention used by real TOTP apps (Google Authenticator etc.):
        // the RFC's own worked example is 8-digit for illustration only,
        // production TOTP (RFC 6238 as deployed) is 6-digit, verified here
        // against a known-good 6-digit reference implementation output.
        let code = generate_totp(RFC6238_SHA1_SEED_B32, 59).unwrap();
        assert_eq!(code, "287082");
    }

    #[test]
    fn matches_rfc6238_test_vector_at_time_1234567890() {
        let code = generate_totp(RFC6238_SHA1_SEED_B32, 1_234_567_890).unwrap();
        assert_eq!(code, "005924");
    }

    #[test]
    fn rejects_invalid_base32_seed() {
        let result = generate_totp("not-valid-base32!!!", 59);
        assert!(matches!(result, Err(TotpError::InvalidSeed)));
    }

    #[test]
    fn code_is_always_six_digits_zero_padded() {
        // Time chosen so the underlying dynamic-truncation value is small
```

```rust
use axum::body::Body;
use axum::extract::Request;
use axum::http::{HeaderName, HeaderValue, StatusCode};
use axum::response::Response;

/// Forward `req` to `{upstream_base}{path_and_query}`, returning the
/// upstream response's status, headers, and body unmodified.
pub async fn forward(
    upstream_base: &str,
    client: &reqwest::Client,
    req: Request,
) -> Response {
    let method = req.method().clone();
    let path_and_query = req
        .uri()
        .path_and_query()
        .map(|pq| pq.as_str())
        .unwrap_or("/");
    let url = format!("{upstream_base}{path_and_query}");

    let mut builder = client.request(
        reqwest::Method::from_bytes(method.as_str().as_bytes()).unwrap(),
        &url,
    );
    for (name, value) in req.headers().iter() {
        if name == axum::http::header::HOST {
            continue; // let reqwest set Host for the upstream URL
        }
        builder = builder.header(name.as_str(), value.as_bytes());
    }

    let body_bytes = match axum::body::to_bytes(req.into_body(), usize::MAX).await {
        Ok(b) => b,
        Err(_) => {
            return Response::builder()
                .status(StatusCode::BAD_GATEWAY)
                .body(Body::from("failed to read request body"))
                .unwrap();
        }
    };
    builder = builder.body(body_bytes.to_vec());

    let upstream_resp = match builder.send().await {
        Ok(r) => r,
        Err(_) => {
            return Response::builder()
                .status(StatusCode::BAD_GATEWAY)
                .body(Body::from("upstream request failed"))
                .unwrap();
        }
    };

    let status = StatusCode::from_u16(upstream_resp.status().as_u16()).unwrap();
    let mut response_builder = Response::builder().status(status);
    for (name, value) in upstream_resp.headers().iter() {
        if let (Ok(name), Ok(value)) = (
            HeaderName::from_bytes(name.as_str().as_bytes()),
            HeaderValue::from_bytes(value.as_bytes()),
        ) {
            response_builder = response_builder.header(name, value);
        }
    }
    let body = upstream_resp.bytes().await.unwrap_or_default();
    response_builder.body(Body::from(body)).unwrap()
}

#[cfg(test)]
mod tests {
    use super::*;
    use axum::body::to_bytes as body_to_bytes;
    use std::io::{Read, Write};
    use std::net::TcpListener;

    /// Minimal single-shot HTTP mock server, same technique as
    /// `provider-hilan`'s `spawn_test_server` (shaon-audit
    /// crates/provider-hilan/src/client.rs:1697) — read one request, reply
    /// with a canned response, then the thread exits.
    fn spawn_single_response_server(response: Vec<u8>) -> (String, std::thread::JoinHandle<()>) {
        let listener = TcpListener::bind("127.0.0.1:0").unwrap();

            let (mut stream, _) = listener.accept().unwrap();

            stream.write_all(&response).unwrap();
        });

    }

        format!(
            "HTTP/1.1 200 OK\r\nContent-Type: text/plain\r\nContent-Length: {}\r\nConnection: close\r\n\r\n{}",
            body.len(),

        .into_bytes()
    }
```
